# Security Policy

## Supported Versions

We actively monitor and maintain security patches only for the current operational release deployed to production.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability—such as an issue with our data collection pipeline, client-side data sanitation, or workflow execution architecture—please do not open a public GitHub Issue. 

Instead, please report it responsibly by contacting the maintainer directly.

We take security issues seriously and will make every effort to acknowledge and remediate verified vulnerabilities swiftly. Thank you for helping keep this project secure!
